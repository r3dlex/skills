# A2A Cross-Agent Handoff Convention

Generated by `init-ai-repo` (ADR-0005). Promotes the single MCP/A2A mention into
a real surface: Agent2Agent (A2A) is the open standard for cross-agent
delegation, the complement to MCP (Model Context Protocol) for tool access.
This document is a **generated convention**, not a live router — it defines the
shape a handoff takes; no network or model call runs at generation time.

## Handoff envelope

Every cross-agent delegation passes a single JSON handoff envelope:

| Field | Meaning |
| --- | --- |
| `correlation_id` | Shared run identifier; preserved end-to-end across the handoff so the trajectory stays one trace (see `.ai/observability/conventions.md`). |
| `from_agent` | The delegating agent role. |
| `to_agent` | The receiving agent role. |
| `task` | The delegated unit of work and its acceptance check. |
| `context_refs` | Pointers (paths, node IDs) to shared context; large context is referenced, not copied. |
| `constraints` | Guardrails the receiver must honor (offline-only, no hosted mutation). |

## Rules

- The receiving agent opens a child span linked by `correlation_id`, preserving
  the trajectory across delegation (mirrors the trace conventions).
- Handoffs reference context by pointer; they never inline secrets or
  credentials into the envelope.
- A handoff carries an explicit acceptance check so the delegator can verify the
  returned result, not assume it.
- The convention is transport-neutral: it describes the envelope, not a wire
  protocol; the MCP registry stub (`.ai/mcp/registry.json`) binds tools, this
  doc binds cross-agent delegation.
- Offline/deterministic: no live A2A endpoint is resolved at generation time.
