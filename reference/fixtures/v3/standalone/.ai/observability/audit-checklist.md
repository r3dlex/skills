# Observability Audit Checklist

Generated by `init-ai-repo` (ADR-0005). Run this checklist when reviewing an
agent run or a PR that changes agent behavior. It is offline and deterministic:
the checklist is a generated convention, not a live CI gate. Token-cost and
trajectory metering execute out-of-band; CI only verifies this checklist exists.

## Token-cost audit

- [ ] Prompt and completion token counts are recorded per agent run.
- [ ] Token cost is attributed to a model-routing tier (`frontier`/`mid`/`cheap`).
- [ ] Cost-per-run is compared against the prior baseline; regressions are flagged.
- [ ] No run silently escalates to a higher tier without a recorded reason.

## Trajectory audit

- [ ] The ordered tool/model/handoff trajectory is captured for the run.
- [ ] The trajectory matches the expected plan; deviations are flagged as drift.
- [ ] No prohibited calls (network, hosted mutation) appear in the trajectory.
- [ ] Cross-agent handoffs preserve the `correlation_id` end-to-end.

## Drift signals

- [ ] Latency and cost trends are reviewed across runs, not only per run.
- [ ] Repeated retries, loops, or abandoned trajectories are surfaced.
- [ ] Any flagged drift is linked to a traceability node for follow-up.
