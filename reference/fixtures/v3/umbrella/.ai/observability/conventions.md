# Observability Conventions

Generated by `init-ai-repo` (ADR-0005). Observability is non-optional harness
surface: without it there is no way to tell whether an agent is doing well or
quietly drifting. These conventions are **generated documentation**, not live
metering — token-cost and trajectory metering run out-of-band, never in CI.

## Logging conventions

- Structured, line-delimited JSON. One event per line; append-only.
- Every event carries `ts` (ISO-8601 UTC), `level`, `agent`, `phase`, `event`,
  and a `correlation_id` shared across a single agent run.
- Levels: `debug`, `info`, `warn`, `error`. Default emit level is `info`.
- Secrets, credentials, and tokens are never logged; redact before emit.
- Logs are local artifacts under `.ai/drift/` or the host runner; they are not
  synced by physical-copy.

## Trace conventions

- A trace is the ordered span tree for one agent run, keyed by `correlation_id`.
- Each span records `span_id`, `parent_span_id`, `name`, `start`, `end`, and a
  `kind` in `{tool-call, model-call, handoff, verification}`.
- Tool and model calls record token counts and latency when the host exposes
  them; absence is recorded as `null`, never fabricated.
- Cross-agent handoffs (A2A) open a child span on the receiving agent linked by
  `correlation_id`, preserving the trajectory across delegation.

## Metering dimensions

- **Token cost** — prompt/completion tokens and derived cost per agent run.
- **Latency** — wall-clock per span and per run.
- **Trajectory** — the ordered tool/model/handoff sequence, audited for drift
  against the expected plan.

Metering values are produced by an out-of-band runner and recorded as evidence;
CI validates only that these conventions and the audit checklist exist.
